рҹ§  How Eaglercraft Multiplayer Works (Simple Version)

The Problem

Eaglercraft cannot connect directly to a Minecraft server because browsers cannot open TCP sockets. They can only use HTTP/HTTPS and WebSocket.

The Solution

Browser (WebSocket) вҶ’ Proxy/Translator вҶ’ Minecraft Server (TCP)

Codespaces runs all three pieces inside one Docker container:

  • Browser вҖ” You playing Eaglercraft in your web browser
  • WebSocket вҖ” Secure connection from browser to Codespaces (wss://)
  • Proxy вҖ” Converts WebSocket packets to Minecraft protocol
  • Minecraft Server вҖ” Spigot or Paper in offline mode

What Happens When You Connect

  1. You open Eaglercraft in your browser
  2. You add your Codespaces URL: wss://PORT-CODESPACE.github.dev
  3. Browser opens WebSocket connection (automatically secure, wss://)
  4. EaglercraftX server receives connection
  5. Translator converts browser packets вҶ’ Minecraft packets
  6. Spigot/Paper server processes them normally
  7. You play Minecraft in your browser, connected to your server

File Persistence

Codespaces stores files on GitHub servers. Your world data is saved in Docker volumes:

  • ./world вҖ” Main world and player data
  • ./world_nether вҖ” Nether dimension
  • ./world_the_end вҖ” End dimension

These persist as long as your Codespace exists (even if you close the browser).

Why This Is Better Than Old Methods

  • No reverse proxy needed вҖ” Codespaces handles it
  • No SSL certificates вҖ” GitHub manages it
  • No domain registration вҖ” You get *.github.dev free
  • No VPS cost вҖ” Free tier includes 120 hours/month
  • Everything in one container вҖ” No juggling multiple services
